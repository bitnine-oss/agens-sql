2.4. 패키지

통합 패키지에 포함된 패키지 목록은 다음과 같다.

  1. pgpool-II

    • PostgreSQL 서버와 PostgreSQL DB 클라이언트 사이에 동작하는 미들웨어.

    • Connection pooling, Load Balancing 기능을 통해 시스템 전체의 처리량을 향상시키고 Replication 기능을 통해 사용자에게 고가용성(High Availability)를 제공한다.

  2. PostGIS

    • PostgreSQL에 지리적 객체기능을 더하여 공간정보 처리기능을 효율적으로 수행할 수 있도록 한다.

    • 이를 이용하여 지리적 객체를 이용한 SQL query문을 작성할 수 있다.

  3. PgBouncer

    • PostgreSQL에 connection pooling 기능을 가능하게 해주는 가벼운 모듈이다.

    • Session pooling을 이용하여 서버와 클라이언트의 연결 상태를 유지하고, 클라이언트가 연결을 해제하면 서버 연결은 pool에 반환된다. 이로써 세션과 서버 간의 연결이 성립될 때 갖게 되는 부하를 줄일 수 있다.

    • Transaction pooling을 이용하여 트랜잭션 동안 서버 연결은 클라이언트에 할당되고, PgBouncer가 해당 트랜잭션이 끝났음을 알리면 서버 connection은 pool에 반환된다.

    • Statement pooling은 PL/Proxy를 이용하는 클라이언트의 “autocommit” 모드를 적용하기 위한 것이다.

  4. PGQ

    • Queueing Solution으로써 트랜잭션에 대한 비동기 batch 작업을 손쉽게 해결하여 준다.

    • 다량의 INSERT/UPDATE/DELETE문이 포함된 트랜잭션에 대한 batch작업 도중, 즉 트랜잭션 내부 commit 이전에 특정 trigger를 수행하고 싶을 경우 트랜잭션의 수행을 막지 않고 원하는 작업을 할 때 매우 유용하게 사용될 수 있다.

  5. PL/Proxy

    • proxy함수를 작성하여 호출하는 방식으로 사용하며, proxy 함수를 수y행하면 원격 데이터베이스 내에 구현되어 있는 함수를 로컬의 proxy 함수 내부에 명시되어 있는 원격 주소만으로도 원격 함수와 동일한 동작을 가능케 한다.

    • proxy함수를 작성하여 호출하는 방식으로 사용하며, proxy 함수를 수행하면 원격 데이터베이스 내에 구현되어 있는 함수를 로컬의 proxy 함수 내부에 명시되어 있는 원격 주소만으로도 원격 함수와 동일한 동작을 가능케 한다.

  6. pg_hint_plan

    • 사용자가 직접 query에 hint절을 추가하여 최적의 query execution plan을 세울 수 있도록 한다.

  7. POWA(POstgresql Workload Analyzer)

    • 다양한 소스로부터 PostgreSQL 서버에 대한 성능 통계를 수집 및 종합하여 실시간으로 차트와 그래프 형태로 제공하는 모듈이다.
